The realtime backend for web apps

Build realtime webapps without the hassle of handling websockets or data synchronisation

The Roadrunner concept

Todays web applications need to respond fast to user input and allow many users to collaborate on the same data. Roadrunner is a approach on how this can be done with modern and standard technologies. At the moment we rely heavily on the Websockets standard and take advantage of the crowing support by the Java platform.

At the moment we support the usage of Roadrunner with:

  • vert.x
  • Netty
  • Tomcat 7.0.x
The concept is, that the server holds a representation of the data in JSON and allows the alteration of this representation from the client. The changes are then distributed to other clients.

Features

Data Accessibility

Data is stored as standard JSON, and the Roadrunner client libraries and REST API provide easy access to that data.

Real-time Synchronization

Roadrunner takes a new approach to the way data is moved around an app. Rather than using a traditional request response model, it works by synchronizing data between devices. Whenever your data changes, all clients are immediately notified. The synchronized data is also persisted, allowing new clients to be immediately updated.

Real-time Queries

Roadrunner allows you to define queries that are executed on the server. Whenever the conent of the query result changes (e.g. data changes that add or remove a elements to the query result) the client gets an update.
